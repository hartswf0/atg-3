3
Core Thesis 90 sec

On Screen

Void Management Architecture

When LLMs manage scenes (constructing content directly), complexity accumulates until coherence collapses.

When LLMs manage voids (preparing possibility spaces that humans instantiate), agency is preserved and outputs become exportable.
Purpose
State the thesis in its full, falsifiable form. This is the slide everything else supports.
Tags
CORE CLAIM
Speaker Notes
"This is the distinction. Scene vs. void. When the AI makes specific content—objects, positions, relationships—it's managing a scene. When the AI prepares an empty structure—dimensions, axes, transformation vocabulary—it's managing a void."

"The rest of this talk is evidence for why this distinction matters."
Key Move
Read the definition verbatim. Let it land. Pause after.

Scene vs. Void: The Distinction 120 sec

On Screen

Scene Management ✗ Void Management ✓
LLM generates specific content LLM prepares empty structure
LLM decides positions, relationships LLM defines dimensions, not content
Each generation adds complexity Complexity stays user-scale

Outputs die with session Structure transfers, exports
Purpose

OPERATIONALIZATION PRACTICAL

Implications for AI Tool Design 90 sec

On Screen

Design Principles
  • Define voids, not scenes
  • Imply, don't saturate
  • Bound complexity
  • Preserve human instantiation
  • Enable export
Anti-Patterns
  • LLM "completing" user's work
  • Unbounded generation
  • Session-dependent outputs
  • Saturation over implication
Purpose
Give actionable takeaways. The audience should be able to apply these to their own tool design.
Tags
ACTIONABLE PRINCIPLES
Speaker Notes
"Left column: what to do. Right column: what to avoid."

"'LLM completing user's work'—this is the trap. It feels helpful. But every completion adds scene complexity."
Key Move
"These are things you can check in your own AI tools tomorrow."
