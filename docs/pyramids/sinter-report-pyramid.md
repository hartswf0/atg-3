## LEVEL 5: OPPOSITION (Counterarguments addressed)

### "Better Engineering Will Solve It"
**Claim:** Larger context windows, better memory will fix scene management.
**Response:** Engineering improvements (RAG, memory hierarchies) ARE void management — they impose bounded structures. They work because they bound, not despite it.

### "External Memory Solves It"
**Claim:** Vector databases enable true scene management.
**Response:** External memory creates distributed voids. Retrieval creates bounded slices. The "scene" never actually exists — only bounded retrievals.

### "Users Prefer Complete Outputs"
**Claim:** Void management is more work.
**Response:** Short-term comfort vs long-term satisfaction. Complete outputs lead to complexity grey. Void management enables sustained creative control.
