## THE BINDING THESIS

The Digital LEGO Ecosystem paper argues:

> The "digital brick" is a **contested object**. To the LEGO Group, it was a marketing tool and industrial necessity. To the fan community, it was **liberation from the scarcity of physical matter**.

This maps directly to the **NSP thesis** in `grounded-theory-paper.md`:

| LEGO Domain | NSP Domain | Principle |
|-------------|------------|-----------|
| LDraw (open text) | Type-safe code | Invalid states unrepresentable |
| LDD (physics engine) | Runtime validation | Invalid overlaps rejected |
| Modulex (1:1:1 cube) | Physical constraints | Dimensional error eliminated BY GEOMETRY |

## CONNECTION TO NSP THESIS

The Digital LEGO Ecosystem **IS** Negative Space Programming in the material domain:

| NSP Concept | LEGO Manifestation |
|-------------|-------------------|
| **Invalid states unrepresentable** | LDU coordinate system (12 LDU = stud diameter) |
| **Types as constraints** | .dat/.ldr/.mpd file hierarchy |
| **Correctness by construction** | LDraw primitives (stud.dat reused) |
| **Ghost evidence** | Deleted LDD code (2022) |
| **Physical NSP** | Modulex 1:1:1 cube eliminated scaling error |

**The LDraw unit is a TYPE.** An invalid position (13.5 LDU stud) is mathematically representable but semantically invalid—the part won't align with other parts. The file format enforces correctness.
