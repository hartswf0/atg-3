01 Toolkit Description

ATG 3.0 (also called COURAGE) is a trail-aware documentation and design toolkit for researcher-practitioners working at the intersection of HCI, AI ethics, and design. It addresses a core challenge in LLM-assisted research: the gap between what the human knows, what the LLM knows, and what remains invisible to both‚Äîwhat we call "void management."

The toolkit consists of five interconnected components:

The toolkit addresses three interrelated problems:

03 Audiences

Primary users: Graduate students and researcher-practitioners conducting design research, particularly those managing complex, evolving projects with LLM assistance. The toolkit assumes intermediate technical literacy (comfort with HTML, JSON, Git) but not programming expertise.

Secondary beneficiaries: Advisors and reviewers who receive better-structured, better-cited research outputs. The Trail Olog provides at-a-glance project comprehension; the Pattern Glossary provides shared vocabulary.

Tertiary stakeholders: Future researchers who may build on this documented trail. The Genome Sequencer ensures projects leave behind legible records.

~100 words

04 Context of Use

This toolkit is designed for use during the research and documentation phase of design projects‚Äîafter initial exploration but before final submission. It assumes researchers have access to their own data trails (commit logs, file systems, notes) and produces structured outputs that feed into academic writing.

The toolkit is explicitly not for real-time design sessions but for reflective pause. It operates in the gap between doing and documenting‚Äîthe "cooling off" period where raw work becomes legible artifact.

Context Usage
Education Teaching research methods and documentation practices
Audit Post-hoc review of project evolution and decisions
Documentation Generating structured appendices and reflection pieces

05 Levers & Leverage Points

Shilton's Values Levers (2013)

Following Shilton's framework for embedding ethics into design, the toolkit operates primarily through reflection levers‚Äîthe POML library's phase-sequencing (PLAN‚ÜíTHINK‚ÜíSPEAK) forces deliberation before action. The Void Guardian prompt embodies the anticipation lever by demanding proof of claims before publication. The Pattern Glossary functions as an integration lever, embedding ethical vocabulary into technical artifacts.

Lever Toolkit Manifestation
Anticipation Void Guardian demands citations before claims
Reflection POML phases force pause before action
Integration Pattern Glossary links ethics to code
Participation Genome lets others read project DNA

Meadows' Leverage Points (via Erlichman)

In Meadows' hierarchy of system intervention, the toolkit intervenes at Level 9 (Information Flows)‚Äîmaking previously hidden trails visible‚Äîand Level 4 (Self-Organization)‚Äîenabling practitioners to create their own patterns and genomes rather than consuming pre-made frameworks. The highest leverage aim is Level 2 (System Goals): shifting from "publish fast" to "publish grounded."

Lessig's Regulatory Modes (2006)

Using Lessig's four modalities of constraint, the toolkit operates primarily through architecture (UI that requires loading data before visualization, phases that must complete in sequence) and norms (making citation auditing a "normal" practice rather than optional polish). It does not invoke law or market mechanisms directly.

06 Design Process

The toolkit emerged through autoethnographic design research over 16 weeks (Fall 2025). The first artifact was documentation (README.md), followed by interactive prototypes, then integration layers. This "documentation-first" approach meant the system's conceptual architecture was articulated before its technical implementation.

Key influences included:

  • Warburg's Mnemosyne Atlas for spatial knowledge representation
  • Nelson's transclusion for trail-aware citation
  • McLuhan's Tetrad for multidirectional analysis
  • Spivak's Ologs for categorical knowledge encoding

Building used single-file HTML architecture for maximum portability‚Äîeach tool is a self-contained document that works offline. Testing was primarily eating one's own cooking: using the toolkit to document its own creation. The POML library emerged from repeated patterns in prompt crafting across 70+ HTML interfaces.

07 Learnings & Reflections

Building this toolkit revealed that constraints are not obstacles but infrastructure. The "void"‚Äîwhat the LLM cannot see, what remains uncited, what the user hasn't loaded‚Äîshapes the system as much as content does. This insight reframed the entire project: instead of minimizing constraints, the toolkit makes constraints legible.

"I initially assumed the LLM should manage the scene; I discovered it should manage the void."

The hardest part was maintaining coherence across 18+ POML files and 70+ HTML interfaces. The Genome Sequencer emerged precisely from this challenge‚Äîa need to see the whole while working on parts. If I started over, I would establish the Genome schema earlier and build outward from it.

Unexpected discovery: the POML library became the most valuable component, not the visualizations. The prompts encode tacit methodological knowledge in reusable form‚Äîa kind of "research method in a file."

08 Future Directions

Future work includes:

  • Validation: User studies with other researcher-practitioners beyond self-use
  • Generalization: Abstracting the POML library for domains beyond HCI
  • Community: Open-sourcing with workshop materials for adoption
  • Integration: Connecting Trail Olog to actual Git history for automated genome extraction
  • Cross-Linking: Enabling genome comparison across sibling projects

The ultimate vision is a research practice where trails are first-class citizens, not afterthoughts‚Äîwhere the void is mapped, not ignored.
